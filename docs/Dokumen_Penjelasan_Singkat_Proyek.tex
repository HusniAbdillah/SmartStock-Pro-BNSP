\subsection{1. PENDAHULUAN \& LATAR
BELAKANG}\label{1-pendahuluan--latar-belakang}

Dokumen ini disusun sebagai bukti pemenuhan seluruh unit kompetensi
dalam skema Web Developer. \textbf{SmartStock Pro} dirancang untuk
mendigitalisasi pengelolaan stok di 5 gudang PT Maju Bersama Digital
yang sebelumnya manual, guna meminimalisir kesalahan data dan
mempercepat proses transfer barang.

\subsection{Ringkasan Implementasi
Aktual}\label{ringkasan-implementasi-aktual}

\begin{itemize}
\item Backend: Laravel Framework 11.54.0, PHP 8.2+
\item Database: MySQL 8.x (\texttt{DB\_CONNECTION=mysql})
\item Queue: Laravel Queue database driver (\texttt{jobs},
  \texttt{failed\_jobs}, \texttt{job\_batches})
\item Frontend: Blade, Tailwind CSS 3.4.19, Vite 6.4.2
\item Chart: Chart.js 4.4.0 via CDN
\item Map: Leaflet 1.9.4 + OpenStreetMap
\item PDF: \texttt{barryvdh/laravel-dompdf} 3.1.2
\item Import: CSV background job \texttt{ProcessBatchImport}; package
  \texttt{maatwebsite/excel} 3.1.69 tersedia sebagai dependency, tetapi flow
  import aktual memakai parser CSV native PHP
\item Notification: Laravel database notification + polling
  \texttt{/api/notifications/unread}
\item Server monitoring: \texttt{/api/server-resources} dan
  \texttt{/api/health}
\end{itemize}

\section{Bab 1 - Pendahuluan}\label{bab-1---pendahuluan}

\subsection{1.1 Latar Belakang}\label{11-latar-belakang}

PT Maju Bersama Digital adalah perusahaan distribusi barang elektronik
yang mengelola 5 gudang di Jakarta, Surabaya, Bandung, Medan, dan
Makassar. Pengelolaan inventaris lama menggunakan spreadsheet
menyebabkan data stok tidak sinkron, transfer barang lambat, laporan
stok membutuhkan 2-3 hari, tidak ada alert stok minimum, dan manajemen
tidak memiliki monitoring inventaris real-time.

SmartStock Pro dibangun untuk menggantikan proses manual tersebut dengan
aplikasi web terpusat berbasis Laravel. Sistem menyediakan pengelolaan
produk, kategori, gudang, supplier, transaksi masuk/keluar, transfer
antar gudang, notifikasi stok minimum, audit log, error log, upload
gambar produk, import CSV, export PDF, dashboard grafik, peta gudang,
dan monitoring resource server.

\subsection{6.7 FAQ}\label{67-faq}

\subsubsection{1. Apa itu SmartStock
Pro?}\label{1-apa-itu-smartstock-pro}

SmartStock Pro adalah website inventaris multi-gudang untuk PT Maju
Bersama Digital, menggantikan spreadsheet manual dengan sistem berbasis
Laravel.

\subsubsection{2. Database apa yang
digunakan?}\label{2-database-apa-yang-digunakan}

Database uji/production menggunakan MySQL 8.x. SQLite hanya default
\texttt{.env.example} bawaan Laravel untuk development cepat.

\subsubsection{3. Apakah mendukung 5
gudang?}\label{3-apakah-mendukung-5-gudang}

Ya. Seeder menyediakan gudang Jakarta, Surabaya, Bandung, Medan, dan
Makassar lengkap dengan koordinat.

\subsubsection{4. Apa saja role
pengguna?}\label{4-apa-saja-role-pengguna}

Admin, Manajer Gudang, Staf Gudang, dan Viewer.

\subsubsection{5. Bagaimana sistem mencegah stok
minus?}\label{5-bagaimana-sistem-mencegah-stok-minus}

Transaksi keluar dan transfer memeriksa stok tersedia sebelum melakukan
decrement.

\subsubsection{6. Apakah transfer antar gudang
aman?}\label{6-apakah-transfer-antar-gudang-aman}

Ya. Transfer memakai \texttt{DB::transaction()} sehingga perubahan stok
asal dan tujuan atomik.

\subsubsection{7. Apakah real-time memakai
WebSocket?}\label{7-apakah-real-time-memakai-websocket}

Tidak. Implementasi aktual memakai polling API berkala: 12 detik untuk
notifikasi dan 8 detik untuk resource monitor.

\subsubsection{8. Bagaimana cara import data
produk?}\label{8-bagaimana-cara-import-data-produk}

Upload file CSV melalui menu Produk. Job \texttt{ProcessBatchImport}
memproses data di background.

\subsubsection{9. Mengapa laporan PDF besar perlu
queue?}\label{9-mengapa-laporan-pdf-besar-perlu-queue}

Agar proses generate PDF tidak membuat UI freeze dan tidak menahan
request utama.

\subsubsection{10. Apakah sistem memakai Redis atau
Celery?}\label{10-apakah-sistem-memakai-redis-atau-celery}

Tidak. Queue aktual memakai Laravel database queue. Redis dapat menjadi
rencana peningkatan jika job meningkat.

\subsubsection{11. Apakah sistem memakai
PostgreSQL?}\label{11-apakah-sistem-memakai-postgresql}

Tidak. Untuk implementasi ini, dokumentasi diselaraskan ke MySQL sesuai
penggunaan Anda.

\subsubsection{12. Bagaimana jika aplikasi
error?}\label{12-bagaimana-jika-aplikasi-error}

Unhandled exception dicatat ke tabel \texttt{error\_logs} dengan
severity \texttt{critical} dan dapat dilihat dari menu Error Log.
